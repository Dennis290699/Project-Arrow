\section{Requerimientos Tecnicos}

\subsection{Requerimientos de desarrollo}

El equipo de desarrollo requiere Unity 6 LTS, un editor de codigo compatible con C\#, Git, Git LFS y acceso al repositorio de GitHub. Se recomienda utilizar una version comun del motor para evitar incompatibilidades en escenas, prefabs y paquetes.

\subsection{Requerimientos funcionales}

El prototipo debe incluir movimiento lateral, salto, combate cuerpo a cuerpo, ataque a distancia, enemigos con IA basica, sistema de salud, recoleccion de Diamantes de Almas, HUD, checkpoints, menus principales y al menos una condicion de victoria o derrota.

\subsection{Requerimientos no funcionales}

El juego debe mantener una tasa de cuadros estable en equipos de gama media, ofrecer controles responsivos, presentar una interfaz legible en resoluciones comunes y conservar una organizacion de proyecto comprensible para el equipo.

\subsection{Configuracion de repositorio}

El archivo \texttt{.gitignore} debe estar configurado para Unity, ignorando carpetas como \texttt{Library}, \texttt{Temp}, \texttt{Obj}, \texttt{Builds} y otros archivos generados localmente. Los activos binarios pesados deben rastrearse mediante Git LFS.

\subsection{Criterios minimos de entrega}

La entrega academica debe incluir ejecutable funcional para Windows, repositorio organizado, documentacion del GDD, evidencia de control de versiones, capturas o video de demostracion y descripcion de riesgos y mitigaciones.
